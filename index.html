History and How to use

[Slide 2.1]
JavaScript, когда создавался изначально, не был предназначен (designed) для больших приложений, в нём была заложена динамическая типизация.
То есть любой пременной всегда можно было присвоить другой тип данных (например была переменная типа "строка", и ей можно было присвоить тип "массив").

За последние годы JavaScript существенно эволюционировал. Теперь, это уже не просто обычный язык для написания скриптов, а (complete programming language) полноценный язык программирования, который служит для создания клиентских приложений, серверных приложений, мобильных и десктопных приложений, используется повсеместно, где он необходим.

So, let's start with the history.

[Slide 2.2]
TypeScript was first introduced in 2012 by Microsoft, and its development was led by Danish programmer - Anders Hejlsberg, the creator of C#.

Anders Hejlsberg

[Slide 2.3]
TypeScript is a superset of JavaScript, meaning it builds upon JavaScript by adding static typing, which provides developers with more robust and predictable code.

[Slide 2.4]
It was designed to address some of the challenges faced by JavaScript developers, such as the lack of static typing, which can lead to errors and make large codebases difficult to maintain. By adding static typing, TypeScript brings the benefits of early error detection, improved tooling support, and better code organization.
